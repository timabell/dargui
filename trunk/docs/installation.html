Chapter 4. Compilation and Installation

    How to install DarGUI

    deb and rpm packages - DarGUI can be downloaded from Sourceforge as Debian or RPM packages which can be installed with the appropriate tools.

    Binary tarball - DarGUI can be downloaded from Sourceforge as a binary archive, compressed using gzip. To install using this method, extract the contents of the archive to a convenient location and then execute the script install.sh. If you should later want to remove DarGUI you can do this by executing the script uninstall.sh.

    Compilation from source - DarGUI is written in Freepascal using the Lazarus Component Library. To compile DarGUI from source you will need to install  Freepascal and Lazarus either from your distribution's package manager or from the Lazarus download page. Note that versions available through your distribution's package manager may not be recent enough to have all the necessary components.
    To compile, startup Lazarus and open the DarGUI project file using Project->Open Project from the main menu. There are two project files in the DarGUI source directory, named dargui.lpi and darlogger.lpi. You will need to open and compile both projects. To compile, select Run->Build all.
    Release version 0.5.2 of DarGUI was compiled using Freepascal 2.4.0 and Lazarus 0.9.29 beta SVN revision 27340.

    Possible compile problems - the most likely problem that the user will encounter when compiling is that the linker will fail with a message similar to
    /usr/bin/ld: cannot find -lvte.
    This means that the linker has been unable to find the library file libvte.so. To  fix it, first check to see if libvte9 is installed on the system , either by checking via your package manager or by entering
    ls /usr/lib/libvte*
    from a terminal. If the resulting output is something like libvte.so.9@  libvte.so.9.13.1 then libvte9 has been installed already and you now have to make a symlink from it to libvte.so by opening a terminal as root and entering
    ln -s /usr/lib/libvte.so.9 /usr/lib/libvte.so    .

    The linker should now be able to link the program with the libvte9 library. This process may be required for some other libraries, depending on how Lazarus was installed.

    Requirements - DarGUI uses xterm and the bash shell to run dar scripts. DarGUI 0.5.2 also requires libvte9. To schedule automatic backups atd and/or cron are required.
